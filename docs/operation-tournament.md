## 1. 概要

### 公開モデル

トーナメントは**作成時点では非公開**（`publicModes: []`）で作成されます。

| 状態 | 一般プレイヤー | 管理者 |
|------|---------------|--------|
| 非公開 (`publicModes: []`) | 一覧に表示されない・詳細アクセス不可 | 全アクセス可 |
| 部分公開（例: `["ta"]`） | TA モードのみ閲覧可・それ以外は非表示 | 全アクセス可 |
| 全公開 (`["ta","bm","mr","gp"]`) | 全モード閲覧可 | 全アクセス可 |

**重要**: 認証済みプレイヤーは非公開トーナメントのAPIに直接アクセスできます（参加者ページで自分のタイム入力等を行うため）。一般公開かどうかとは独立した概念です。

### 公開の順序制約

モードは以下の順序でのみ公開できます（スキップ不可）:

```
TA → BM → MR → GP
```

例えば、BM を公開するには先に TA を公開済みである必要があります。

## 2. トーナメント作成フロー

### 2.1 トーナメント作成

1. 管理者ダッシュボードの **「+ 新しいトーナメント」** をクリック
2. 大会名・日付を入力して **「作成」** をクリック
3. 作成されたトーナメントは **非公開状態** で始まります

### 2.2 前日準備（プレイヤー登録・エントリー）

各モードのプレイヤー設定は**大会前日**に行います:

1. 対象モードのページ（例: `/tournaments/:id/ta`）を開く
2. **「プレイヤー設定」（Setup Players）** ダイアログを開く
3. 参加プレイヤーを全員追加して **「保存」** をクリック
4. 参加者ページ（`/participant`）のURLをプレイヤーに共有し、ログイン方法を案内する

> プレイヤーは非公開トーナメントでもログイン後にタイム入力などが可能です。前日にエントリーを完了させておくことで、当日の入力をスムーズにできます。

## 3. モード公開ワークフロー

### 3.1 公開操作

1. 任意のトーナメントモードページを開く（例: `/tournaments/:id/ta`）
2. タブバー直下のコントロールエリアに **「[モード名]: 公開」** ボタンが表示される
3. ボタンをクリックすると即座にタブの「未公開」バッジが消え、一般公開状態になる

### 3.2 当日の公開手順（推奨）

```
大会開始の発表と同時に:
1. TA ページで「タイムトライアル: 公開」ボタンをクリック
2. BM ページで「バトルモード: 公開」ボタンをクリック（TA 公開後）
3. MR ページで「マッチレース: 公開」ボタンをクリック（BM 公開後）
4. GP ページで「グランプリ: 公開」ボタンをクリック（MR 公開後）
```

> 各モードは前のモードが公開済みでないと公開できません（順序制約）。

### 3.3 未公開状態に戻す（緊急時）

同じボタンを再度クリックすると、そのモードおよびそれ以降のモードが未公開に戻ります。

例: BM を未公開に戻すと、MR・GP も自動的に未公開になります。

## 4. プレイヤー管理

### 4.1 プレイヤー作成

1. 管理者ダッシュボード → **「プレイヤー管理」**
2. **「+ プレイヤー追加」** をクリック
3. 名前・ニックネーム・国を入力して登録
4. 発行された**仮パスワード**をプレイヤーに安全な方法で伝達

### 4.2 パスワードリセット

プレイヤーがパスワードを忘れた場合:
1. プレイヤー管理ページで対象プレイヤーの **「パスワードリセット」** をクリック
2. 新しい仮パスワードを発行してプレイヤーに伝達

## 5. 当日運用チェックリスト

### 大会前日

- [ ] トーナメント作成済み
- [ ] 全プレイヤーのアカウント作成・パスワード配布済み
- [ ] 各モードのプレイヤーエントリー設定済み（Setup Players 保存済み）
- [ ] プレイヤーへの参加者ページURL共有済み

### 大会当日（開始前）

- [ ] 各モードページを開いてエントリーデータ確認
- [ ] TA 凍結状態の確認（タイム入力受付中であること）

### 大会当日（開始時）

- [ ] 発表と同時に TA → BM → MR → GP の順でモードを公開
- [ ] 公開後、タブの「未公開」バッジが消えていることを確認
- [ ] プレイヤーへの大会開始アナウンス

### 大会終了後

- [ ] 最終スコア・ランキングの確認
- [ ] 必要に応じてトーナメントの status を `completed` に更新

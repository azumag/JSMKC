1. PRをチェックし、コンフリクトがあればそれをファイル編集をして解決し、再度pushせよ
2. PRをチェックし、レビューがついていれば、それをファイル編集をして解決し、再度pushせよ
3. Mainブランチに移動し、Mainブランチを最新化せよ
4. github で main に取り込まれた内容を調べて、それをE2Eシナリオとして既存シナリオに追加し、テストも追加し、commit&pushすること。そのあとE2Eテストを実行し、もし途中でfailしたらそのfailした部分を修正し、commit&pushすること。fail理由がテストでなくプロダクトコード側にあると判断した場合は、プロダクトコードを直して、それをdeployした上で再度e2eを実行すること。修正が終わったら、修正した部分以外の全体のE2Eを実行して問題がないか確かめること。問題があれば適宜修正しcommit&pushし、deployされるのを5分待ち、再度確認すること。大事なポイント：E2Eテストは本番にむいているので、テスト側でなくロジック側に変更を加えた場合はdeployしなければ確認できない。
5. 全ての動作が終わったら1時間待機し、再度1から実行せよ

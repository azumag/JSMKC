# TAバトルロワイヤル運用仕様

## 1. 競技形式の選択

トーナメント作成時に、TAの競技形式を次の2種類から選択する。

- **通常TA**: 予選 → Phase 1 → Phase 2 → Phase 3。Phase 3は3ライフで、残り8人・4人・2人の時点でライフを回復する。
- **TAバトルロワイヤル**: 予選とPhase 1/2を使用せず、参加者設定後すぐPhase 3を開始する。初期10ライフ、ライフ回復なし、個別ハンデあり。

バトルロワイヤルを選択した場合は、作成前の確認ダイアログでルールと変更ロックを確認する。Phase 3の参加者またはラウンドが作成された後は、APIを直接呼んでも競技形式を変更できない。

TAバトルロワイヤル大会のナビゲーションにはTAのみを表示し、BM・MR・GP・総合モードは管理者を含めて表示しない。配信管理は競技モードではないため、管理者向けタブとして維持する。

## 2. 開始条件と進行

バトルロワイヤルの開始画面で2人以上を選択し、大会内ハンデを確定する。「Phase 3開始」を実行すると、予選エントリを経由せず、選択した参加者のPhase 3エントリを直接作成する。各参加者には10ライフと設定した大会内ハンデを保存する。

開始後にTAトップを開いた場合は、Phase 3決勝画面へ自動的に移動する。参加者追加・削除と大会内ハンデ変更は開始時点でロックされる。

各ラウンドでは補正後タイムの昇順で順位を決め、参加者数の下位半分がライフを失う。奇数人数の場合、上位側が1人多くなる。

- 通常TAの8人・4人・2人ライフ回復は適用しない。
- 通常TAの失格人数上限やrevival判定は適用しない。
- 同じラウンドで複数人が0ライフになった場合、対象者を全員失格にする。
- 0ライフかつ未失格の状態を残さない。
- 最後の1人が優勝者となる。

順位境界の同タイムや3位決定条件など、既存のサドンデス判定は維持する。

### 2.1 ラウンド別ライフ減少量

下位半分が失うライフ数は既定1だが、管理者はラウンド開始時に1〜9の範囲で個別に設定できる（`TTPhaseRound.lifeLoss`、既定値1）。この設定は通常TAには存在せず、TAバトルロワイヤルのPhase 3ラウンドのみで有効。

- 決勝画面の「ラウンド開始」操作に「このラウンドのライフ減少量」セレクトが表示される（バトルロワイヤルのみ）。既定は1で、ラウンド開始のたびに1へリセットされる（一時的な上書きであり、恒久設定ではない）。
- `POST /api/tournaments/:id/ta/phases` の `start_round` アクションが任意の `lifeLoss`（1〜9の整数）を受け取る。通常TAまたはPhase 1/2に対して1以外の値を指定した場合は400エラー（`INVALID_LIFE_LOSS`）を返す。
- 設定したラウンドの下位半分は、その`lifeLoss`の値だけライフを失う。サドンデス（同タイム・トップ4同時ラストライフ）が発生した場合も、元となったラウンドの`lifeLoss`が最終的な失格判定に使われる。
- ラウンド履歴・送信前確認ダイアログには実際に適用される減少量（例:「（ライフ-2）」）を表示する。
- undo・キャンセル・アーカイブ再生などのライフ再構築処理も、ラウンドごとの`lifeLoss`を参照する（既定1へのフォールバック付き）。

## 3. 大会内ハンデ

使用できるハンデは `0秒`、`-1秒`、`-3秒`、`-5秒` の4種類である。

- `TTEntry.taHandicapSeconds`: 唯一の設定箇所。大会参加者画面（予選エントリ追加時・バトルロワイヤル開始時）で参加者ごとに設定し、Phase 3開始時にその値を使用する。
- ラウンド結果の `handicapSeconds`: そのラウンドで実際に適用した監査値。

新規参加者は常にハンデ`0`で追加され、大会参加者画面（`manageTournamentHandicaps` / バトルロワイヤル開始画面）で個別または一括で変更する。以前存在した「選手マスタの既定ハンデ」（`Player.taHandicapSeconds`）は、プレイヤー管理画面で設定できるように見えて実際には大会エントリに一切反映されない紛らわしい項目だったため廃止した（プレイヤー管理では設定できず、必ず大会参加者画面で設定する）。

Phase 3開始前は開始画面で参加者とハンデを編集できる。Phase 3エントリが作成された後は、参加者とハンデを変更できない。

## 4. タイム計算とリトライ

通常入力では次の式で補正する。

```text
補正後タイム = max(0, 生タイム + ハンデ秒 × 1000)
```

例: 生タイム104,000ms、ハンデ-5秒の場合、補正後は99,000msとなる。

リトライは既存の固定ペナルティタイムを使い、ハンデを適用しない。保存結果は次を含む。

- `rawTimeMs`: 入力またはリトライ固定値。
- `handicapSeconds`: 実際の適用値。非適用時は0。
- `timeMs`: 順位決定に使用した補正後値。
- `isRetry`: リトライかどうか。
- `tvNumber`: 使用TV番号。未指定はnull。

結果送信前の確認ダイアログと保存後のラウンド履歴は、同じ共通計算・正規化関数を使用する。

## 5. APIと表示

`POST /api/tournaments/:id/ta/battle-royale` は管理者専用の開始APIである。2人以上の参加者と大会内ハンデを受け取り、Phase 3エントリを直接作成する。通常TA大会、重複参加者、存在しない選手、開始済み大会は拒否する。

`GET /api/tournaments/:id/ta/phases` はライブ・アーカイブともに次を返す。

- `taMode`: `standard` または `battle_royale`
- `taBattleRoyaleMode`: 既存クライアント互換用boolean
- `phase3Rules.initialLives`
- `phase3Rules.lifeResetThresholds`
- `phase3Rules.survivorsNeeded`
- `phase3Rules.handicapEnabled`
- `phase3Rules.retryAppliesHandicap`

各ラウンドのレスポンス（`rounds[].lifeLoss`）にはそのラウンドで実際に適用されたライフ減少量を含む（既定1、§2.1参照）。

決勝画面はこのレスポンスだけでルールを表示する。残ライフはハートを10個並べず、`♥ 7/10` の形式で表示する。履歴はバトルロワイヤル時に生タイム、ハンデ、補正後タイムを表示する。

`rounds[].results[]` には `livesAfter`（そのラウンド終了直後の残りライフ）と `lifeLost`（そのラウンドでライフを失ったか）がサーバー側で付与される。ラウンド履歴の各行では、ライフを失った選手（下位半分・敗退を含む）の横に `残り{lives}` として表示され、ダメージを受けた選手の残りライフを他ラウンドまで遡らずに確認できる。上位半分（ダメージなし）の選手には表示しない。`livesAfter`/`lifeLost` はラウンド単体の値ではなく、`results`・`eliminatedIds`・`livesReset`・各ラウンドの`lifeLoss`（§2.1）・確定済みサドンデス結果の履歴全体をラウンド番号順に再生して算出するため、TTEntry.lives（現在値のみ保持）を経由せず過去ラウンドの値も正しく復元できる（`src/lib/ta/phase3-life-replay.ts`）。ライブ・アーカイブ双方のレスポンスに同じロジックで付与される（ただしアーカイブは確定済みサドンデス結果を保持しないため、境界タイの判定は生タイム順にフォールバックする）。

## 6. アーカイブ

新規アーカイブはschema version 2で保存し、競技形式、確定ルール、大会内ハンデ、生タイム・ハンデ・補正後タイムを含む。

schema version 1は通常TAとして正規化する。

- 初期3ライフ
- 回復閾値8/4/2
- ハンデ無効
- 欠損した生タイムは`timeMs`
- 欠損したハンデは0

アーカイブ再生では、保存済みエントリ状態とラウンドの`eliminatedIds`・`livesReset`を優先し、現在の選手マスタや大会設定を参照しない。

## 8. 手動受け入れ確認

1. 通常TAとバトルロワイヤルを作成画面で比較できる。
2. バトルロワイヤル作成前に確認ダイアログが出る。
3. バトルロワイヤル大会ではTAと配信管理だけが表示され、BM・MR・GP・総合が表示されない。
4. 開始画面で2人以上を選択し、大会内ハンデを設定できる。
5. 開始操作だけでPhase 3エントリが直接作成され、予選画面やPhase 1/2が表示されない。
6. Phase 3開始後にTAトップを開くと決勝画面へ移動し、参加者・ハンデ変更が拒否される。
7. 10ライフ、回復なし、補正後順位が画面に表示される。
8. 6人中下位3人が同時に0ライフとなった場合、3人全員が失格する。
9. リトライへハンデが適用されない。
10. ライブ削除後もアーカイブで同じルール・残ライフ・結果を再現できる。
11. 通常TAの予選、3ライフ、8/4/2回復、ハンデなしが維持される。
12. 決勝画面で「このラウンドのライフ減少量」に2を設定してラウンドを開始すると、下位半分のライフが2減少し、履歴に反映される（§2.1）。通常TAではこのセレクトが表示されない。

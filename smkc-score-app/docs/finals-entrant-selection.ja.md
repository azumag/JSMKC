# 決勝進出者の選び方（2・3・4グループ）

対象: `selectFinalsEntrantsByGroup()` と、Top 24からバラージ・Top 16を生成する配置ロジック。

## 1. 共通の進出枠

決勝進出枠の人数はグループ数によらず共通である。

```text
perGroup = 12 / グループ数

各グループ 1〜perGroup 位                 → 直接進出（合計12人）
各グループ perGroup+1〜2*perGroup 位      → バラージ（合計12人）
各グループ 2*perGroup+1 位以降            → 敗退
```

各グループには最低 `2 * perGroup` 人が必要である。グループ内順位は、各モードの順位規則と、必要な場合の直接対決・サドンデス確定順位を反映した最終順位を使う。

進出人数は共通だが、選手をどのシード枠へ配置するかはグループ数によって異なる。

## 2. 2グループ: 固定配置

`perGroup = 6`。各グループ1〜6位が直接進出、7〜12位がバラージへ進む。グループAとBの成績を横断比較して配置を動かしてはならない。

2グループでは均等分散と同一グループ同士の1回戦回避を両立できるため、手書きの固定配置マップを使用する。

### 2.1 直接進出者

```text
Upper seed → グループ内順位
1:A1  2:B3  3:B1  4:A3  5:B2  6:A4
7:A2  8:B4  9:A5  11:B5  13:B6  15:A6
```

Upperの残り `10 / 12 / 14 / 16` はバラージ勝者用の枠である。

### 2.2 バラージ

表示シード13〜24に次の固定順で配置する。

```text
13:B8  14:B7  15:A8  16:A7  17:B9  18:A11
19:B10 20:A12 21:A10 22:B12 23:A9  24:B11
```

バラージ1回戦とUpper進出先は次の通り。

```text
[23,22] の勝者 vs [13] → Upper 16
[19,18] の勝者 vs [16] → Upper 12
[17,20] の勝者 vs [15] → Upper 14
[21,24] の勝者 vs [14] → Upper 10
```

この配置は勝点・得失点差・シードランキングによって変化しない。

## 3. 3グループ: 動的バケット方式

`perGroup = 4`。各グループ1〜4位が直接進出、5〜8位がバラージ、9位以降は敗退となる。

3グループでは固定マップだけで均等分散を作れないため、同じグループ内順位の選手をバケットにまとめ、バケット内だけを次の順で動的に並べる。

1. WDLスコア／勝点（降順）
2. 得失点差またはドライバーズポイント（降順）
3. 完全同点なら、グループ横断サドンデスで確定した `combinedRankOverride`

シードランキングはタイブレークに使わない。バケットNの全選手は、バケットN+1の全選手より必ず上位になる。

バケット1〜4をシード1〜12、バケット5〜8をシード13〜24として連番採番する。CDM 2025公式結果で確認された配置は次の通り。

```text
Upper R1: [1,16] [8,9] [4,13] [5,12]
          [2,15] [7,10] [3,14] [6,11]

Barrage R1: [17,24] [20,21] [18,23] [19,22]
Barrage BYE: 16 / 13 / 15 / 14
```

3グループではバラージ勝者が自身のシード13〜16を保持したままUpperへ進む。同一グループ同士の1回戦が発生する場合がある。

## 4. 4グループ

4グループでは均等分散できるため、2グループと同様に固定配置マップを使用する方針である。ただし正式マップは未提示であり、現在のUI/APIから4グループ大会は作成できない。

内部には暫定的なバケット処理が残っているが、正式仕様ではない。大会運営から固定マップが提示されるまでは本番機能として有効化しない。

## 5. CDM Excelエクスポート

BM/MRの決勝ブラケット生成後のエクスポートは、グループ数に対応した配置からシードリストと試合スロットを復元する。

- 2グループ: 本書2章の固定配置を使用する。
- 3グループ: 本書3章の動的順位とCDM 2025配置を使用する。
- GP Finalsのシード一覧はExcel数式を維持するため、予選順位が異なる場合は完全一致を保証できない。
- 予選シートの順位表はExcelが独自に再計算するため、直接対決・サドンデス確定順・不均等グループを含む場合はJSMKC表示と異なることがある。
